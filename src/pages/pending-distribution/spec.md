# 待我配货

## 📋 业务与功能

### 1.1 核心目标
待我配货页面用于展示需要当前登录用户处理的配货需求列表。用户可以通过筛选条件快速定位待处理的配货单，查看详情并进行配货操作（调拨或采购）。

### 1.2 功能清单

- **[面包屑导航]**：显示当前页面位置（需求申请/待我配货），提供导航功能
- **[筛选查询]**：支持多维度筛选，包括需求单号-行号、实际需求人、采购类别、采购单品、需求部门、需求提单人、需求产生原因、预计效果收益、提交时间范围、处理人
- **[列表展示]**：展示所有待配货数据，支持分页、批量选择
- **[去配货]**：点击打开配货抽屉，支持选择配货方式（调拨/采购）
- **[配货方式切换]**：调拨或采购，采购方式下显示采购相关表单字段

### 1.3 交互要点

- [筛选展开/收起]：点击展开/收起按钮可展开或收起更多筛选项
- [搜索]：点击搜索按钮根据当前筛选条件查询数据
- [还原]：点击还原按钮清空所有筛选条件
- [去配货]：点击去配货按钮打开右侧抽屉，进行配货操作
- [配货方式切换]：在抽屉中切换调拨/采购方式，显示不同表单内容
- [批量操作]：勾选列表行可进行批量操作

## 🎨 布局与结构

### 3.1 整体布局

- **布局模式**：单栏布局，内容区居中
- **容器宽度**：流式宽度，最大 1440px
- **关键尺寸**：
  - 页面内边距：24px
  - 筛选区与表格间距：16px
  - 表格行高：48px
  - 配货抽屉宽度：屏幕宽度的 1/3（约 480px）

### 3.2 响应式适配

- **桌面端（≥1200px）**：完整展示所有列
- **平板端（768-1199px）**：隐藏部分次要列（预算相关、项目相关）
- **移动端（<768px）**：简化布局，操作按钮收起为下拉菜单

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `onDistributeClick` | `string` | 点击去配货按钮时触发 | 传递当前行数据 JSON 字符串 |
| `onSearch` | `string` | 点击搜索按钮时触发 | 传递筛选条件 JSON 字符串 |
| `onReset` | `string` | 点击还原按钮时触发 | 传递事件标识 |

| 动作名称 | Params 类型 | 参数说明 | 功能描述 |

| `setFilter` | `string` | JSON 字符串 {"field": "字段名", "value": "值"} | 设置筛选条件 |

### 5.3 变量列表（varList）

| 变量名称 | 类型 | 默认值 | 说明 |
|---------|-----|-------|------|
| `selected_rows` | `array` | `[]` | 当前选中的行数据 |
| `filter_conditions` | `object` | `{}` | 当前筛选条件 |
| `drawer_visible` | `boolean` | `false` | 配货抽屉是否可见 |
| `distribute_method` | `string` | `"transfer"` | 配货方式（transfer/purchase） |

### 5.4 配置项列表（configList）

| 配置项名称 | 类型 | 默认值 | 说明 |
|----------|-----|-------|------|
| `pageTitle` | `string` | `"待我配货"` | 页面标题 |
| `defaultHandler` | `string` | `"张三(san.zhang)"` | 默认处理人 |
